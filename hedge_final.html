· Beta(基金/PSI20) = 0.6271（R²=79%，基金跟踪PSI20较好）
· Beta(基金/IBEX) = 0.423（R²=42%，IBEX只能解释基金42%的波动）

注：R²=42%意味着基金有58%的波动无法被IBEX解释。Put只对冲IBEX相关的那42%风险。
条件Beta（急跌时）：上面的Beta=0.423是全样本（含平时+急跌）的平均值。 但实证显示，在11次急跌事件中，基金跌幅/IBEX跌幅的平均比率为0.524， 比全样本Beta高24%。
这意味着：急跌时基金对IBEX的真实敏感度更高。按Beta=0.423计算的16张合约， 在急跌时实际只能覆盖约81%的IBEX相关损失，而非理论上的100%。
学术背景：危机中跨市场相关性上升是公认现象（Longin & Solnik 2001）。条件Beta > 无条件Beta是正常的。 但10次样本量偏小（比率范围0.369~0.702），不宜过度精确化。 本报告仍以全样本Beta定合约数，但提醒用户实际覆盖率可能低于理论值。

七、操作步骤（方案A）

  1. IBKR账户：开通欧洲期权交易权限，交易所选MEFF。
  2. 买第一腿——ATM Put ×8：搜索Mini IBEX期权，到期月2027年3月，类型Put，行权价17,350（ATM，50点间距）。参考价约€1,056/张，8张合计约€8,450。
  3. 买第二腿——90%OTM Put ×20：同到期月，行权价15,600（90% OTM）。参考价约€419/张，20张合计约€8,388。
  4. 总保费约€16,838（Black-Scholes理论值，IV=18.5%，r=2.6%）。 实际市价预计上浮10-30%，尤其OTM Put因波动率偏斜（skew）真实IV约22-25%，比报告使用的平值IV=18.5%更高， OTM部分实际价格可能高于BS理论值30-50%。下单前务必以IBKR/MEFF实际报价为准。
  5. 分腿动态滚仓（关键！）：ATM腿和OTM腿职责不同，滚仓策略也不同：
    ATM×8（跟踪腿）：IBEX涨超10%（>19,074）时立即滚仓——卖掉旧ATM Put，买入新ATM Put重设行权价。 ATM必须紧贴当前市场，否则小跌时赔不了（Event #10教训）。建议每月检查。
    OTM×20（兜底腿）：不参与动态滚仓，只做年度正常到期滚仓。 OTM买来就是防崩盘（-20%~-30%），IBEX涨10%后它从虚值10%变成虚值20%，但真来大崩盘时仍会深度实值，赔付差额有限。
  6. 到期前1个月滚仓：ATM和OTM两条腿都正常到期滚仓，卖旧买新，周而复始。
分腿滚仓的成本优势：每次动态触发只滚8张ATM（而非全部28张），大幅降低滚仓损耗。
· 全部滚仓（旧方案）：28张全滚，单次损耗 ≈ 8×€1,056×40% + 20×€419×40% = €6,735，每年2次 = €13,470/年
· 分腿滚仓（推荐）：只滚8张ATM，单次损耗 ≈ 8×€1,056×40% = €3,380，每年2次 = €6,760/年
· 每年节省约€6,711，5年节省约€33,553

代价：在"连续大涨后崩盘"的极端场景中，未滚仓的OTM行权价较低，赔付会少约€34,679。 但这笔差额与5年节省的滚仓成本大致打平。在更常见的"涨后中小跌"场景中，OTM本来就不赔，分不分开滚没有区别。

七、操作步骤（方案B）

  1. IBKR账户：开通欧洲期权交易权限，交易所选MEFF。
  2. 买90%OTM Put ×24：搜索Mini IBEX期权，到期月2027年3月，类型Put，行权价15,600（90% OTM，50点间距）。参考价约€419/张，24张合计约€10,066。
  3. 总保费约€10,066（Black-Scholes理论值，IV=18.5%，r=2.6%）。 实际市价预计上浮30-50%——OTM Put因波动率偏斜（skew）真实IV约22-25%，远高于平值IV=18.5%。 实际年成本可能达€14,092~15,099。下单前务必以IBKR/MEFF实际报价为准。
  4. 动态滚仓：方案B全部是OTM Put，与方案A的分腿滚仓逻辑类似—— OTM的职责是防崩盘，IBEX涨10%后从虚值10%变成虚值20%，但大崩盘时仍会深度实值。 因此方案B以年度正常滚仓为主，不需要频繁动态触发。
    但如果IBEX累计涨超20%（>20,808），OTM行权价已严重脱离市场，此时应滚仓重设行权价。建议每月检查。
  5. 到期前1个月滚仓：正常到期前卖旧买新，周而复始。
方案B注意：因为全部是OTM Put，小幅回调时Put不会赔付。这是刻意的选择——用更低成本换取"只防大灾"的保护。 如果你发现自己担心5-10%的回调没有保护，应该切换到方案A。

方案B的滚仓成本优势：因为以年度滚仓为主（动态触发阈值为20%，远高于方案A的10%）， 预计每年额外动态滚仓0-1次，滚仓损耗远低于方案A。 即使触发1次：24×€419×40% = €4,026。 方案B真实年化总成本 ≈ €10,066 + €2,013（平均0.5次/年） = €12,079（未含skew上浮）。

八、局限性（必读）

Event #10 教训：年度滚仓的致命缺陷
2025年3-4月，基金跌7.4%（约€49,000），这是持仓期最大一次回撤。但12月ATM Put的行权价设在2024年7月买入时的IBEX水平（约11,090点，注意：这是历史回测值，当前推荐行权价为17,350点）， 到事件发生时IBEX已涨到13,484点，即使跌到11,786点仍在行权价之上——Put几乎是废纸，覆盖仅约5%。

结论：固定年度滚仓在"先涨后跌"行情下保护形同虚设。这就是为什么操作步骤中加入了动态滚仓触发（IBEX涨超10%即提前滚仓重设行权价），并建议每月检查。
  1. 这是减震垫，不是全额保险：方案A的覆盖率随跌幅递增（小跌~24%，大跌可达~57%），方案B仅在IBEX跌超10%后才启动赔付。在先涨后跌场景下可能远低于此。即使加入动态滚仓，也无法保证覆盖率。
  2. 保费是确定支出：每年€16,838（方案A），5年不出事白花€84,190（组合的13.1%）。
  3. R²=42%的根本限制：IBEX只能解释基金42%的波动。基金可能因为葡萄牙本地原因（个股暴雷、流动性危机）大跌而IBEX无动于衷，此时Put完全无效。
  4. 条件Beta高于无条件Beta：全样本Beta=0.423用于计算合约数，但11次历史急跌中实际比率平均0.524（高24%）。 这意味着16张合约在急跌时只能覆盖约81%的IBEX相关损失， 实际保护效果比场景表显示的更弱。样本量仅11次，比率波动大（0.37~0.70），结论存在不确定性。
  5. 线性模型在极端行情下失真：场景表使用恒定Beta，但极端尾部事件中Beta会漂移，覆盖率可能偏离预期。
  6. IV影响成本：恐慌期Put更贵，尽量在平静期滚仓。

九、总结

方案A：ATM ×8 + 90%OTM ×20，12个月年滚 + 动态滚仓
能做到的：历史11次急跌IBEX 100%同步下跌。混合配置用同样预算换更多合约，大跌时赔付高于纯ATM×16，同时8张ATM保留小跌保护。
做不到的：无法覆盖IBEX以外58%的风险（R²=42%）。在"先涨后跌"行情下，如果没有及时动态滚仓，Put可能接近废纸。
成本：年化2.61%（€16,838/年），5年约€84,190，是确定的支出。
本质：这是一个减震垫，不是全额保险。它降低了系统性暴跌中的最大亏损幅度，但不能保证你不亏钱。
方案B：纯90%OTM ×24，12个月年滚 + 动态滚仓
能做到的：在崩盘式暴跌（>10%）时提供赔付，成本比方案A低40%。
做不到的：5-10%的中等回调完全没有保护。同样无法覆盖IBEX以外58%的风险。
成本：年化1.56%（€10,066/年），5年约€50,330。
适合：能承受中等回调、只想防黑天鹅的投资者。省下的保费本身也是一种保护。
