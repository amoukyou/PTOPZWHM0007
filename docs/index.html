条件Beta（急跌时）：上面的Beta=0.423是全样本（含平时+急跌）的平均值。 但实证显示，在11次急跌事件中，基金跌幅/IBEX跌幅的平均比率为0.524， 比全样本Beta高24%。
这意味着：急跌时基金对IBEX的真实敏感度更高。按Beta=0.423计算的16张合约， 在急跌时实际只能覆盖约81%的IBEX相关损失，而非理论上的100%。
学术背景：危机中跨市场相关性上升是公认现象（Longin & Solnik 2001）。条件Beta > 无条件Beta是正常的。 但10次样本量偏小（比率范围0.369~0.702），不宜过度精确化。 本报告仍以全样本Beta定合约数，但提醒用户实际覆盖率可能低于理论值。

七、操作步骤（方案A）

  1. IBKR账户：开通欧洲期权交易权限，交易所选MEFF。
  2. 买第一腿——ATM Put ×8：搜索Mini IBEX期权，到期月2027年3月，类型Put，行权价17,350（ATM，50点间距）。参考价约€1,051/张，8张合计约€8,410。
  3. 买第二腿——90%OTM Put ×20：同到期月，行权价15,600（90% OTM）。参考价约€417/张，20张合计约€8,338。
  4. 总保费约€16,749（Black-Scholes理论值，IV=18.5%，r=2.6%），实际市价可能上浮10-30%。
  5. 动态滚仓触发（关键！）：不要死等12个月到期。IBEX涨超15%（>19,954）时必须提前滚仓—— 卖掉旧Put（已变深度虚值），买入新的ATM Put重设行权价。这能防止"先涨后跌"时Put变废纸（Event #10教训）。
  6. 到期前1个月滚仓：如果IBEX没有大涨，正常到期前卖旧买新，周而复始。
动态滚仓会增加交易频率和额外保费支出（每次滚仓损失旧Put的剩余时间价值），但能确保行权价始终贴近当前市场， 避免保护失效。预计每年触发0-2次额外滚仓。

八、局限性（必读）

Event #10 教训：年度滚仓的致命缺陷
2025年3-4月，基金跌7.4%（约€49,000），这是持仓期最大一次回撤。但12月ATM Put的行权价设在2024年7月买入时的IBEX水平（约11,090点，注意：这是历史回测值，当前推荐行权价为17,350点）， 到事件发生时IBEX已涨到13,484点，即使跌到11,786点仍在行权价之上——Put几乎是废纸，覆盖仅约5%。

结论：固定年度滚仓在"先涨后跌"行情下保护形同虚设。这就是为什么操作步骤中加入了动态滚仓触发（IBEX涨超15%即提前滚仓重设行权价）。
  1. 这是减震垫，不是全额保险：在理想线性假设下覆盖约46%的基金损失。在先涨后跌场景下可能远低于此。即使加入动态滚仓，也无法保证覆盖率。
  2. 保费是确定支出：每年€16,749（方案A），5年不出事白花€83,745（组合的13.0%）。
  3. R²=42%的根本限制：IBEX只能解释基金42%的波动。基金可能因为葡萄牙本地原因（个股暴雷、流动性危机）大跌而IBEX无动于衷，此时Put完全无效。
  4. 条件Beta高于无条件Beta：全样本Beta=0.423用于计算合约数，但11次历史急跌中实际比率平均0.524（高24%）。 这意味着16张合约在急跌时只能覆盖约81%的IBEX相关损失， 实际保护效果比场景表显示的更弱。样本量仅11次，比率波动大（0.37~0.70），结论存在不确定性。
  5. 线性模型在极端行情下失真：场景表使用恒定Beta，但极端尾部事件中Beta会漂移，覆盖率可能偏离预期。
  6. IV影响成本：恐慌期Put更贵，尽量在平静期滚仓。

九、总结

方案A：ATM ×8 + 90%OTM ×20，12个月年滚 + 动态滚仓
能做到的：历史11次急跌IBEX 100%同步下跌。混合配置在大跌时赔付比纯ATM多33-60%，同时保留小跌基本保护。
做不到的：无法覆盖IBEX以外58%的风险（R²=42%）。在"先涨后跌"行情下，如果没有及时动态滚仓，Put可能接近废纸。
成本：年化2.60%（€16,749/年），5年约€83,745，是确定的支出。
本质：这是一个减震垫，不是全额保险。它降低了系统性暴跌中的最大亏损幅度，但不能保证你不亏钱。
